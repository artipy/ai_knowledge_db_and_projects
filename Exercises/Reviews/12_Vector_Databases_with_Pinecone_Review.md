## Детальная оценка по частям

### Часть 1: Создание индекса — 12/15 баллов

#### ⚠️ Задача 1.1: Неверная размерность индекса (–2 балла)

Индекс создан с `dimension=1024` вместо `dimension=1536`, как требует задание (модель `text-embedding-3-small`). Это следствие использования локальной модели вместо OpenAI API.

```python
# Фактически:
pc.create_index(name='news-index', dimension=1024, ...)

# Должно быть:
pc.create_index(name='news-index', dimension=1536, ...)
```

Похвально, что перед `create_index` вызывается `pc.delete_index('news-index')` — это хорошая практика идемпотентности при повторных запусках ноутбука. Список индексов выведен корректно.

#### ⚠️ Задача 1.2: Формат вывода стоимости (–1 балл)

Функция `create_article_text` реализована правильно — форматирует все три поля корректно. Подсчёт токенов через tiktoken верен (263 токена). Однако стоимость выводится в научной нотации вместо десятичной:

```python
# Фактически:
print(f'Cost: ${cost}')   # → Cost: $5.2600000000000005e-06

# Должно быть (как в шаблоне):
print(f'Cost: ${cost:.6f}')  # → Cost: $0.000005
```

### Часть 3: Загрузка векторов — 15/20 баллов

#### ⚠️ Задача 3.1: Использование локальной модели вместо OpenAI API (–3 балла)

Задание явно требует `from openai import OpenAI` и `client.embeddings.create(model='text-embedding-3-small', ...)`. Вместо этого используется кастомная функция `create_embeddings()`, работающая через локальный URL и локальную модель:

```python
# Фактически:
client_emb = OpenAI(api_key=os.getenv('LOCAL_TOKEN'), base_url=os.getenv('LOCAL_URL'))

def create_embeddings(texts):
    response = client_emb.embeddings.create(model=os.getenv('LOCAL_MODEL'), input=texts)
    ...

# Должно быть:
client = OpenAI(api_key=os.getenv('OPENAI_API_KEY'))
response = client.embeddings.create(input=article_texts, model='text-embedding-3-small')
```

Концептуальное понимание верное, но требование задания явно работать с OpenAI API не выполнено.

#### ⚠️ Задача 3.1: Отдельные вызовы вместо батча (–2 балла)

Эмбеддинги создаются по одному в list comprehension — 10 отдельных API-вызовов вместо одного батчевого:

```python
# Фактически (10 вызовов):
vectors = [
    {'id': article['id'], 'values': create_embeddings(create_article_text(article))[0]}
    for article in articles
]

# Эффективнее (1 вызов):
response = client.embeddings.create(input=article_texts, model='text-embedding-3-small')
vectors = [
    {'id': articles[i]['id'], 'values': response.data[i].embedding}
    for i in range(len(articles))
]
```

Проверка размерности (`all(...)`) выполнена корректно — паттерн правильный, только значение 1024 вместо 1536.

#### ✅ Задача 3.2: Загрузка через upsert

`index.upsert(vectors=vectors)` вызван корректно. Проверка `total_vector_count == 10` пройдена:

```

# Фактический порядок (неотсортированный):

art-06 | score=0.3541  ← должен быть последним
```

Pinecone должен возвращать результаты отсортированными — это нестандартное поведение, вероятно связанное с особенностями локальной модели или namespace-запросов.
