# Операции с векторами в Pinecone

Продолжение темы [[OpenAI_API/Vector_Databases_with_Pinecone]]. Здесь рассматриваются операции получения, запроса, обновления и удаления векторов.

Связанные темы: [[OpenAI_API/Vector_Databases_with_ChromaDB]] · [[OpenAI_API/Working_with_Embeddings]] · [[OpenAI_API/Embeddings_Applications]]

## Получение векторов (Fetching)

**Fetch** — поиск и возврат записи по её уникальному ID. Аналог поиска книги по ISBN: вы знаете точный идентификатор и хотите получить конкретную запись.

```python
index.fetch(ids=['0', '1'])
```

Ответ содержит:
- `'vectors'` — словарь с записями для каждого запрошенного ID (значения вектора + метаданные)
- `'usage'` — количество потреблённых единиц чтения (Read Units)

### Fetch из конкретного пространства имён

```python
index.fetch(
    ids=['0', '1'],
    namespace='namespace1'
)
```

---

## Единицы чтения (Read Units, RU)

**Read Units** — мера Pinecone для оценки ресурсов, затрачиваемых на операции чтения. Расходуются при:
- `fetch` — 1 RU за каждый пакет из 10 записей
- `query` — зависит от числа записей в namespace и размера записей (размерность + объём метаданных)
- `list` — также расходует RU

Pinecone предоставляет справочное руководство по потреблению RU для различных конфигураций. Бесплатный план имеет щедрую ежемесячную квоту.

---

## Запросы к векторам (Querying)

**Query** — отправка вектора в индекс и получение `top_k` наиболее семантически близких векторов. Аналог поиска книг, тематически близких к уже имеющейся.

Применяется в:
- **Чат-ботах**: промпт пользователя → эмбеддинг → поиск релевантного контекста в БД (RAG)
- **Семантических поисковых системах**: запрос → эмбеддинг → ближайшие результаты

```python
index.query(
    vector=[0.123987123, 0.123876, ...],  # вектор той же размерности, что индекс
    top_k=3
)
```

Ответ содержит ID и **score** (оценку сходства) для каждого найденного вектора.

### Возврат значений вектора

По умолчанию значения векторов не возвращаются. Чтобы включить их:

```python
index.query(
    vector=[0.123987123, 0.123876, ...],
    top_k=3,
    include_values=True
)
```

---

## Метрики расстояния

Метод `query` определяет ближайшие векторы, вычисляя расстояние между вектором запроса и векторами в индексе.

| Метрика | Описание |
|---|---|
| `'cosine'` | Косинусное сходство — угол между векторами. Используется по умолчанию. Наиболее распространённая метрика. |
| `'euclidean'` | Евклидово расстояние — расстояние по прямой между двумя точками. |
| `'dotproduct'` | Скалярное произведение. |

Метрика задаётся при **создании** индекса и не может быть изменена впоследствии:

```python
pc.create_index(
    name='datacamp-index-1',
    dimension=1024,
    metric='dotproduct',
    spec=ServerlessSpec(cloud='aws', region='us-east-1')
)
```

> Выбор оптимальной метрики может потребовать экспериментов.

## Фильтрация по метаданным

Фильтрация ограничивает пространство поиска только релевантными записями, сокращая задержку при большом количестве векторов.

Типы поддерживаемых метаданных: строки, числа, логические значения (`bool`), списки строк.

```python
# Пример метаданных
{
    "genre": "action",
    "year": 2020,
    "color": "blue",
    "price": 29.99,
    "is_jeans": True,
    "areas": ["London", "Kent", "Bath"]
}
```

### Базовая фильтрация

Используется параметр `filter` в методе `.query()`:

```python
index.query(
    vector=[-0.250919762305275, ...],
    filter={
        "year": 2019,                       # краткая форма: точное равенство
        "genre": {"$eq": "documentary"}     # явный оператор равенства
    },
    top_k=1
)
```

Оба варианта реализуют фильтрацию по равенству. Форма со словарём и оператором нужна для применения других операторов.

### Операторы фильтрации

| Оператор | Значение |
|---|---|
| `$eq` | Равно |
| `$ne` | Не равно |
| `$gt` | Больше |
| `$gte` | Больше или равно |
| `$lt` | Меньше |
| `$lte` | Меньше или равно |

```python
# Фильтр: year > 2019
index.query(
    vector=[-0.250919762305275, ...],
    filter={
        "year": {"$gt": 2019}
    },
    top_k=1,
    include_metadata=True  # включить метаданные в ответ
)
```

## Обновление векторов

Обновление гарантирует актуальность данных в динамичных средах и поддерживает целостность данных.

### Обновление значений вектора

```python
index.update(id='1', values=[0.09812398, ...])
```

Длина списка должна совпадать с размерностью индекса.

### Обновление метаданных

```python
index.update(
    id='1',
    set_metadata={'genre': 'comedy', 'rating': 5}
)
```

> Указанные поля перезаписываются, остальные поля метаданных остаются без изменений.

### Одновременное обновление вектора и метаданных

```python
index.update(
    id='1',
    values=[0.9871239, ...],
    set_metadata={'genre': 'comedy', 'rating': 5}
)
```

---

## Удаление векторов

Удаление устаревших и неиспользуемых векторов оптимизирует производительность и снижает задержку запросов.

### Удаление по ID

```python
index.delete(ids=['1', '2'])
```

### Удаление по фильтру метаданных

```python
index.delete(
    filter={'genre': 'comedy'}
)
```

### Очистка пространства имён

```python
index.delete(delete_all=True, namespace='namespace1')
```

> Очистка пространства имён удаляет **само пространство имён**. Используйте `delete_all` с осторожностью.
