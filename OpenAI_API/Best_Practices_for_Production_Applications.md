## Модерация контента

Модерация — это процесс анализа входных данных с целью определения нарушений заранее определённых правил. Это защищает пользователей и систему от вредоносного контента.

### Использование специализированных моделей модерации

Часть провайдеров (например, Groq) предоставляют специально обученные модели для классификации текста как `safe` / `unsafe`. Такие модели лучше понимают контекст и избегают ложных срабатываний:

```python
from openai import OpenAI
from dotenv import load_dotenv
import os

load_dotenv()

client = OpenAI(
    api_key=os.getenv('GROQ_TOKEN'),
    base_url=os.getenv('BASE_URL_GROQ')
)

# Текст с упоминанием "смерти" — но в контексте настольной игры
text = '''In the deck of cards are some Exploding Kitten.
When that happens, that person explodes. They are now dead.
This process continues until there's only 1 player left, who wins the game.
'''

moderation_response = client.chat.completions.create(
    model=os.getenv('GROQ_MODEL_MODERATION'),
    messages=[{'role': 'user', 'content': text}]
)

print(moderation_response.choices[0].message.content)
# → safe  (модель понимает игровой контекст)
```

> **Ключевой вывод**: Контекст имеет значение. Одни и те же слова без контекста могут быть классифицированы как опасные, с полным контекстом — как безопасные.

### Ограждения (Guardrails)

Для ограничения тематики приложения используются ограждения — системные сообщения, запрещающие выход за рамки допустимых тем. Они реализуются через `system` роль в запросе:

```python
system_prompt = '''Your role is to assess whether the user question is allowed or not.
The allowed topic is chess only.
If the topic is allowed, reply with an answer as normal,
otherwise say 'Apologise, but the topic is not_allowed'.'''

response = client.chat.completions.create(
    model=os.getenv('GROQ_MODEL'),
    messages=[
        {'role': 'system', 'content': system_prompt},
        {'role': 'user', 'content': text}  # текст про другую игру
    ]
)
# → "Apologise, but the topic is not_allowed."
```

**Стратегии снижения рисков инъекций промптов:**
- Ограничение длины пользовательского ввода
- Ограничение числа генерируемых выходных токенов
- Использование ответов из проверенного набора материалов вместо свободной генерации

## Валидация модели

Валидация — тестирование производительности модели для выявления областей, в которых модель склонна к ошибкам.

### Типичные проблемы, выявляемые валидацией

- Неверное толкование контекста
- Усиление предвзятости из обучающих данных
- Вывод устаревшей информации
- Восприимчивость к манипуляциям для генерации неэтичного контента
- Непреднамеренное раскрытие конфиденциальной информации

### Антагонистическое тестирование (Adversarial Testing)

Метод тестирования, при котором модели предоставляются промпты, специально разработанные для выявления слабых мест:

```python
# Пример: отзыв, начинающийся позитивно, но заканчивающийся негативно
review = '''Это было потрясающе снова увидеть моих любимых актеров спустя 30 лет.
Они выглядят довольно прекрасно. Но на этом все. Они не раскрыли никаких героев,
никакой сюжетной линии. Я даже не переживаю за героев или о том что они делают.'''

response = client.chat.completions.create(
    model=os.getenv('GROQ_MODEL'),
    messages=[
        {'role': 'system', 'content': 'Классифицируй отзыв: Позитивный, нейтральный или негативный.'},
        {'role': 'user', 'content': review}
    ]
)
# Хорошая модель ответит: Негативный
# (несмотря на позитивное начало, общий тон — разочарование)
```

**Сложные случаи для тестирования:**
- Отзывы, меняющие тональность по ходу текста
- Саркастические высказывания (модели часто не распознают сарказм)
- Неоднозначные формулировки

### Стандартизированные наборы для оценки

Исследователи разрабатывают библиотеки с наборами данных для измерения производительности моделей в разных областях (STEM, гуманитарные науки, социальные науки) с уровнями сложности от элементарного до профессионального.

## Лучшие практики безопасности (Safety Best Practices)

### Контентная безопасность

| Мера | Описание |
|------|----------|
| Модерация API | Автоматическая фильтрация небезопасного контента |
| Антагонистическое тестирование | Проверка устойчивости к нежелательным входным данным |
| Ограничение токенов | Предотвращение инъекций и злоупотреблений |
| Ограждения (guardrails) | Управление темой и тоном ответов через system prompt |
| Человеческий контроль | Проверка результатов экспертами в специализированных областях |

### Идентификация пользователей

Для отслеживания нарушений и предотвращения злоупотреблений рекомендуется передавать уникальные идентификаторы пользователей в запросах:

```python
import uuid

# Для зарегистрированных пользователей — хеш имени пользователя
# Для незарегистрированных — UUID сессии
unique_id = str(uuid.uuid4())

response = client.chat.completions.create(
    model=os.getenv('GROQ_MODEL'),
    messages=[{'role': 'user', 'content': 'hi'}],
    user=unique_id  # параметр для идентификации вызова
)
```

### Безопасность API-ключей

- **Хранить на сервере**: никогда не передавать ключи в клиентские среды (браузер, мобильные приложения)
- **Не коммитить в репозитории**: использовать переменные окружения даже в приватных репозиториях
- **Использовать Key Management Services**: для централизованного контроля доступа
- **Регулярно ротировать ключи**: мониторинг использования и своевременная замена

### Дополнительные меры

- Требовать регистрацию и авторизацию пользователей для доступа к сервису
- Предоставить пользователям механизм отчётности о проблемах
- Информировать пользователей об ограничениях модели для правильного формирования ожиданий
