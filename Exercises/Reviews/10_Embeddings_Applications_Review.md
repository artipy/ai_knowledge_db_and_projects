#### ⚠️ Остался: баг с переменной из внешней области видимости (–3 балла)

```python
def classify_article(article, topics, articles):
    classes = [topic['description'] for topic in topics]
    classes_embeddings = create_embeddings(texts)  # 'texts' из внешней области, должно быть 'classes'
```

Функция объявляет локальную переменную `classes`, но при создании эмбеддингов передаёт `texts` — переменную из области видимости ноутбука (задача 5.2). Код работает корректно только потому, что `texts` и `classes` содержат одинаковые значения в текущем контексте выполнения. При вызове функции в другом окружении или после переопределения `texts` поведение будет непредсказуемым.

Исправление:
```python
classes_embeddings = create_embeddings(classes)  # использовать локальную переменную
```
