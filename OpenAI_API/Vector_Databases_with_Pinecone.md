# Векторные базы данных с Pinecone

Pinecone — ведущее полностью управляемое (fully managed) решение для векторных баз данных, предназначенное для создания и масштабирования приложений на базе генеративного ИИ. Как и другие векторные БД, она использует эмбеддинги для хранения и поиска данных на основе семантического сходства. Типичные сценарии применения: чат-боты, поисковые системы, рекомендательные системы и другие ИИ-сервисы.

Связанные темы: [[OpenAI_API/Vector_Databases_with_ChromaDB]] · [[OpenAI_API/Working_with_Embeddings]] · [[OpenAI_API/Embeddings_Applications]] · [[LLMOps/Introduction_to_LLMOps_and_Ideation_Phase]]

## Архитектура Pinecone

Экосистема Pinecone имеет иерархическую структуру:

```
Организации (Organizations)
└── Проекты (Projects)
    └── Индексы (Indexes)
        └── Пространства имён (Namespaces)
            └── Записи (Records / Vectors)
```

**Индексы** — основной компонент. Каждый индекс хранит векторы, обрабатывает поисковые запросы и другие операции. Можно представить их как ящики в картотечном шкафу: каждый ящик — отдельный набор данных.

### Типы индексов

| Тип | Описание |
|---|---|
| **Serverless** | Автоматическое масштабирование, векторы хранятся в объектном хранилище облачной платформы (например, AWS). Дешевле в большинстве случаев. Доступен в бесплатном плане. |
| **Pod-based** | Выбираются предварительно настроенные аппаратные блоки (поды). Контроль над объёмом хранилища, задержкой и пропускной способностью. |

## Создание индекса

```python
from pinecone import Pinecone, ServerlessSpec
from dotenv import load_dotenv
import os

load_dotenv()

# Инициализация клиента
pc = Pinecone(api_key=os.getenv('PINE_API'))

# Создание serverless-индекса
pc.create_index(
    name='datacamp-index',
    dimension=1536,      # размерность векторов (должна совпадать с моделью эмбеддингов)
    spec=ServerlessSpec(
        cloud='aws',
        region='us-east-1'
    )
)
```

Параметр `dimension` должен соответствовать размерности используемой модели эмбеддингов (например, `text-embedding-3-small` возвращает 1536-мерные векторы).

### Просмотр существующих индексов

```python
pc.list_indexes()
```

## Управление индексами

### Подключение к индексу

```python
index = pc.Index('datacamp-index')
```

> Если попытаться подключиться к несуществующему индексу — вернётся ошибка 404.

### Статистика индекса

```python
index.describe_index_stats()
# Возвращает:
# {
#   'dimension': 1536,
#   'index_fullness': 0.0,
#   'total_vector_count': 0,
#   'namespaces': {},
#   ...
# }
```

Поля ответа:
- `dimension` — размерность индекса (должна совпадать с установленной при создании)
- `index_fullness` — степень заполненности индекса
- `total_vector_count` — количество векторов
- `namespaces` — статистика по пространствам имён

> Обновление статистики может занять некоторое время после загрузки данных.

### Пространства имён (Namespaces)

Пространства имён — контейнеры для разделения индекса на сегменты. Полезны для:
- изоляции разных наборов данных
- версионирования данных
- сегментации пользователей или групп

Запросы и операции можно ограничить конкретным пространством имён.

### Роли в организации

| Роль | Права |
|---|---|
| **Organization owner** | Полный доступ к организации: биллинг, пользователи, все проекты |
| **Organization user** | Ограниченные права на уровне организации. Назначается в конкретные проекты как **project owner** |

### Удаление индекса

```python
pc.delete_index('datacamp-index')
```

> Удаление индекса **безвозвратно** уничтожает все содержащиеся в нём записи.

## Загрузка векторов (Vector Ingestion)

### Структура данных

Pinecone требует, чтобы векторы были представлены в виде списка словарей с обязательным полем `id` и `values`:

```python
vectors = [
    {
        "id": "0",
        "values": [0.025525547564029694, ..., 0.0188823901116848]
    },
    {
        "id": "9",
        "values": [0.020712468773126602, ..., 0.006418442353606224]
    },
]
```

### Проверка размерности перед загрузкой

```python
vector_dims = [len(vector['values']) == 1536 for vector in vectors]
all(vector_dims)  # True — все векторы подходящей размерности
```

Если размерность не совпадёт с индексом — API вернёт ошибку.

### Метод upsert

**Upsert** = **up**date + in**sert**: если вектор с таким ID уже существует — обновляется, если нет — создаётся новая запись.

```python
index.upsert(vectors=vectors)
```

### Загрузка векторов с метаданными

Метаданные хранятся в словаре под ключом `"metadata"`. Позволяют фильтровать поиск по дополнительным полям, что значительно ускоряет запросы:

```python
vectors = [
    {
        "id": "0",
        "values": [0.025525547564029694, ..., 0.0188823901116848],
        "metadata": {"genre": "productivity", "year": 2020}
    },
    ...
]

index.upsert(vectors=vectors)
```

> Фильтрация по метаданным позволяет искать только среди релевантных записей, не просматривая весь индекс.

## Сравнение Pinecone и ChromaDB

| Критерий | Pinecone | ChromaDB |
|---|---|---|
| Тип | Managed SaaS | Open-source |
| Масштабируемость | Автоматическая (serverless) | Требует настройки |
| Настройка | API-ключ, регистрация | Локальная установка |
| Стоимость | Платный (есть бесплатный план) | Бесплатный |
| Production-готовность | Высокая | Требует инфраструктуры |
