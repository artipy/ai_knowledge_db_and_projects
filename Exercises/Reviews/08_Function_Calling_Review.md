## Рекомендации для улучшения

### 1. Строгая типизация параметров функций

Всегда проверяйте, ожидается ли число или строка, и указывайте это в `type`. Модель будет возвращать правильный тип:

```python
# Числовые параметры:
'years_experience': {
    'type': 'integer',
    'description': 'Количество лет опыта (только цифра, например: 4)'
}

# Если нужна точность описания:
'price_usd': {
    'type': 'number',
    'description': 'Цена в долларах США (число, без символа $)'
}
```

### 2. Защитный доступ к `tool_calls`

Оборачивайте доступ к `tool_calls` в проверку `finish_reason`:

```python
def safe_extract(response):
    if response.choices[0].finish_reason == 'tool_calls':
        return json.loads(response.choices[0].message.tool_calls[0].function.arguments)
    return None  # или обработать случай отсутствия вызова
```

### 3. Итерация по `tool_calls` при нескольких функциях

```python
results = {}
for call in response.choices[0].message.tool_calls:
    results[call.function.name] = json.loads(call.function.arguments)

print("=== ИНФОРМАЦИЯ О СОБЫТИИ ===")
print(results.get('extract_event_info'))

print("=== ФИНАНСОВАЯ ИНФОРМАЦИЯ ===")
print(results.get('extract_financial_info'))
```

### 4. Соответствие сигнатуры функции требованиям задания

Перед написанием кода прочитайте точные имена параметров из условия. В бонусном задании `get_product_info` явно требовала `product_name` **и** `category`:

```python
'properties': {
    'product_name': {'type': 'string', 'description': 'Полное название продукта'},
    'category': {'type': 'string', 'description': 'Категория товара'}
}
```
