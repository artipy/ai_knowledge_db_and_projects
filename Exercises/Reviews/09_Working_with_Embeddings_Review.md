## Рекомендации для улучшения

### 1. Правильная методология t-SNE

t-SNE — это метод снижения размерности, который преобразует **весь набор данных** в единое двумерное пространство. Применение отдельных трансформаций к подмножествам данных делает координаты несопоставимыми:

```python
import numpy as np
from sklearn.manifold import TSNE
import matplotlib.pyplot as plt

# Собираем все эмбеддинги в одну матрицу
embeddings_matrix = np.array([product['embedding'] for product in products])
labels = [product['category'] for product in products]
names = [product['name'] for product in products]

# Одна трансформация для всех товаров
# perplexity должна быть меньше числа точек (8 товаров → perplexity=3)
tsne = TSNE(n_components=2, perplexity=3, random_state=42)
embeddings_2d = tsne.fit_transform(embeddings_matrix)

# Визуализация с единой системой координат
category_colors = {'Электроника': 'blue', 'Обувь': 'red', 'Книги': 'green'}
for i, (name, label) in enumerate(zip(names, labels)):
    color = category_colors[label]
    plt.scatter(embeddings_2d[i, 0], embeddings_2d[i, 1], c=color, s=100)
    plt.annotate(label, (embeddings_2d[i, 0], embeddings_2d[i, 1]))
```

Только в таком виде можно визуально убедиться, что точки одной категории расположены ближе друг к другу.

### 2. Прототипы категорий для классификации

Вместо классификации по ближайшему соседу (отдельному товару) используйте **прототип категории** — усреднённый вектор всех товаров в категории. Это снижает влияние выбросов и даёт более стабильный результат:

```python
def build_category_prototypes(products):
    from collections import defaultdict
    category_embeddings = defaultdict(list)
    for product in products:
        category_embeddings[product['category']].append(product['embedding'])
    return {
        category: np.mean(embeddings, axis=0).tolist()
        for category, embeddings in category_embeddings.items()
    }

prototypes = build_category_prototypes(products)

def classify_by_prototype(product_name, prototypes):
    query_embedding = create_embeddings(product_name)[0]
    return min(
        prototypes.keys(),
        key=lambda cat: cosine_distance(query_embedding, prototypes[cat])
    )
```

```python
    ('Как приготовить борщ?', False),        # должен вернуть fallback
]

for threshold in [0.1, 0.2, 0.3, 0.5]:
    correct = 0
    for query, should_find in test_queries:
        answer = find_answer(query, faq, threshold=threshold)
        found = answer != 'Извините, я не могу ответить...'
        if found == should_find:
            correct += 1
    print(f'threshold={threshold}: {correct}/{len(test_queries)} корректных')
```

Такой подход позволяет выбрать оптимальное значение порога на основе данных, а не интуиции.
