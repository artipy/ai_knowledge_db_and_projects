### Часть 1: Создание коллекции и загрузка данных — 17/20 баллов

#### ✅ Задача 1.1: Инициализация ChromaDB

`PersistentClient` создан корректно, коллекция `news_articles` создана с функцией эмбеддингов, `list_collections()` вызван. Вывод совпадает с ожидаемым:

#### ✅ Задача 1.2: Функция формирования текста и оценка стоимости

Функция `create_article_text` корректно форматирует три поля:

```python
def create_article_text(article):
    return f'''Headline: {article['headline']}
Topic: {article['topic']}
Keywords: {article['keywords']}'''
```

Подсчёт токенов через tiktoken и расчёт стоимости выполнены верно.

#### ⚠️ Задача 1.3: Несоответствие documents и create_article_text (–3 балла)

В документы коллекции добавляются только заголовки статей, а не полный текст из `create_article_text`:

```python
# Фактически:
documents = [article['headline'] for article in articles]

# Должно быть (согласно логике задания):
documents = [create_article_text(article) for article in articles]
```

Это создаёт противоречие: в Задаче 1.2 реализована функция обогащения текста и рассчитана стоимость эмбеддинга полных текстов (324 токена), но в коллекцию кладутся только заголовки. Фактическая стоимость значительно ниже расчётной, а семантическое качество поиска снижается — ChromaDB не видит поля `Topic` и `Keywords` при поиске.

#### ✅ Задача 2.2: Несколько запросов одним вызовом (исправлено)

Баг с индексацией расстояний устранён — теперь используется `results['distances'][i][j]`:

```python
for i, id in enumerate(reference_ids):
```
